### 一、记录效率 — 最核心的痛点

**问题：记录负担是宝妈放弃育儿App的第一原因。**

当前状态：有Quick Record Sheet和母乳计时器，但仍有改进空间。

| 改进项 | 原因 |
|--------|------|
| **智能默认值** | 凌晨3点打开喂奶记录，应自动选择"亲喂"而非让妈妈重新选；上次喂左边，这次应默认提示右边 |
| **一键记录** | 换尿布时双手脏，至少需要能在Widget上一键完成"尿了"/"拉了"，不需要打开App |
| **语音记录** | 抱着孩子根本腾不出手，"嘿，记一下宝宝刚喂了15分钟奶" |
| **通知栏快捷操作** | 常驻通知条，展开就能点"喂奶/换尿布/入睡"，比打开App快得多 |
| **自动计时联动** | 点了"开始喂奶"，到宝宝睡着后点"入睡"，应自动结束喂奶记录 |
| **批量补录** | 忙了一天忘记录，晚上需要能快速补录一天的记录而不是一条条添加 |

### 二、数据解读 — 从"记录"到"理解"

**问题：原始数据没有洞察，就是无用的负担。**

这是当前App最大的缺失。妈妈真正想知道的不是"今天喂了8次"，而是"这正常吗？"

| 改进项 | 原因 |
|--------|------|
| **"这正常吗？"指标** | 每个数据旁边显示同月龄参考范围。喂奶8次/天 → 显示"0-3月龄正常范围：8-12次" |
| **趋势分析** | "本周比上周平均多睡了42分钟" "喂奶间隔正在拉长，从2小时变为3小时" — 这些规律妈妈自己很难总结 |
| **异常提醒** | 24小时没有喂奶记录时主动提醒；连续3天睡眠低于月龄正常值时柔性提示 |
| **作息规律发现** | 自动发现"宝宝通常下午2点困"、"晚上8-9点是哭闹高峰期"，帮妈妈预判 |
| **阶段性报告** | 满月/百天/半岁自动生成报告："过去30天，宝宝吃了X次奶，睡了Y小时，换了Z片尿布，体重增长了W克" |
| **就医摘要** | 去看儿保时一键生成"最近一个月数据摘要"，直接给医生看 |

---

### 三、数据安全 — 妈妈最怕的是丢数据

**问题：纯离线+无备份 = 手机坏了/丢了，所有记录全没。**

这是当前架构最致命的风险。一年的用心记录，换手机就没了。

| 改进项 | 原因 |
|--------|------|
| **本地备份/恢复** | 至少能导出一个完整备份文件（包含照片），换手机时能导入恢复 |
| **自动备份到本地存储** | 定期自动把数据库文件备份到Download目录或指定位置 |
| **可选的云备份** | 支持备份到用户自己的坚果云，不经过你的服务器，仍然隐私安全 |
| **导入功能** | 当前只有CSV/PDF导出，没有导入。导出的数据无法用来恢复 |

---

### 四、多看护人协作 — 育儿不是一个人的事

**问题：当前App只支持单设备单人使用。**

现实中，爸爸、奶奶、外婆、月嫂都可能帮忙带娃。妈妈上班后，奶奶在家带娃却没法记录。

| 改进项 | 原因 |
|--------|------|
| **多设备同步** | 爸爸手机上也能记录，数据自动合并（可以通过局域网同步而不依赖云端） |
| **看护人标记** | 每条记录标注"谁记的"，妈妈下班后能看到"奶奶今天喂了4次" |
| **交接摘要** | 换班时自动生成"你离开后发生了什么"的摘要 |

---

### 五、情感设计 — 疲惫中的温暖

**问题：当前App是纯工具，缺乏情感连接。**

宝妈在育儿初期情绪波动大，一个每天使用的App如果能带来一点温暖，会产生强烈的情感绑定。

| 改进项 | 原因 |
|--------|------|
| **里程碑庆祝** | 宝宝满100天、第一次翻身、第一颗牙 — 弹出温馨祝贺动画 |
| **成长回忆** | "一年前的今天，宝宝刚出生第3天，你记录了8次喂奶" — 回忆功能 |
| **鼓励文案** | "今天已经喂了6次奶，辛苦了妈妈" — 不是统计数字，而是情感认可 |
| **月龄指南** | 进入第4个月时，主动展示"4月龄宝宝的特点和注意事项" |
| **辛苦可视化** | 累计喂奶时间、换尿布次数的年度总结 — 让妈妈看到自己的付出 |

### 六、阶段适配 — 不同月龄需求完全不同

**问题：当前App对所有阶段展示相同界面。**

| 阶段 | 核心需求 | 当前缺失 |
|------|---------|---------|
| **新生儿(0-3月)** | 喂奶频次、黄疸观察、脐带护理 | 缺黄疸追踪、脐带护理记录 |
| **4-6月** | 开始添加辅食、翻身训练 | 辅食过敏追踪不够系统 |
| **6-12月** | 辅食为主、出牙、爬行学步 | 缺出牙记录、辅食食材库 |
| **1-2岁** | 语言发展、行为习惯 | 缺词汇量追踪、行为记录 |
| **2-3岁** | 如厕训练、入园准备 | 完全缺失 |

App应该根据宝宝当前月龄，**自动调整首页布局和推荐记录类型**。新生儿期突出喂奶和睡眠，6个月后突出辅食和发育里程碑。

### 七、具体功能改进

#### 7.1 喂奶功能
- **左右乳切换提醒**：上次喂的左边，这次提醒"该喂右边了" — 这是每个哺乳妈妈都有的痛点
- **奶量趋势**：瓶喂的奶量变化趋势图
- **辅食过敏追踪**：新食材引入后标记3天观察期，到期提醒"XX没有过敏反应，可以继续"

#### 7.2 睡眠功能
- **睡眠模式分析**：区分白天小睡和夜间长睡，分别统计
- **夜醒次数追踪**：这是妈妈最关心的指标之一
- **入睡方式记录**：奶睡/抱睡/自主入睡 — 追踪入睡方式的改善

#### 7.3 大便功能
- **拍照记录**：大便颜色用选择不够准确，应该支持拍照，并对比标准色卡
- **大便频次异常提醒**：3天没有大便记录时提醒

#### 7.4 生长曲线
- **BMI曲线**：除了体重身高，增加BMI指标
- **生长速度**：不只是绝对值，还要看增速是否正常（"这个月体重增长800g，正常范围500-1000g"）
- **预测曲线**：基于现有数据的趋势预测

#### 7.5 疫苗管理
- **自费疫苗推荐**：除了国家免疫规划，增加常见自费疫苗（如手足口、流感）
- **接种反应记录**：打完疫苗后记录反应（发烧、红肿等），下次接种前回顾
- **补种提醒**：如果错过了某个疫苗，提示补种方案

### 八、UI/交互改进

| 改进项 | 原因 |
|--------|------|
| **深色模式自动切换** | 可以设置晚上自动切换深色模式 — 夜奶时不刺眼 |
| **大字体/大按钮模式** | 给奶奶/外婆用时，她们看不清小字 |
| **手势操作** | 左滑删除、下拉刷新、长按编辑 — 减少操作步骤 |
| **首页自定义** | 让妈妈选择首页显示哪些模块，隐藏不需要的 |
| **记录模板** | 每天重复性高的记录（如固定时间的喂奶），支持一键复制昨天的 |
